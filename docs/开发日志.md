#### 今日总结

利用AI可以很快速的搭建起项目的框架，但是其中存在一个问题

假如编程人员本身对项目代码或是技术栈不够熟悉，将代码生成的工作全部交给ai的话

很快就会出现超过自己知识范围内的代码，这个时候就需要编码人员做出取舍

一种是将编码完全交给AI，拥抱vibe coding，这种解决方法的优缺点都很明显

vibe coding的速度是极快的，基本上只要ai的水平在线，很快就能按照自己的要求生成对应的代码

但是带来的结果是代码的白盒测试可能根本无法进行下去，程序员只能通过黑盒测试再交由AI自己修改

但是AI修改自己的代码有可能会引入新的bug，最终产生的可能是一个完全没法维护升级的巨大屎山代码

另一种是反复审查学习AI生成的代码，好处是AI写的程序员能够理解，本身也是对程序员编码水平的一种提升

但是坏消息是，这种方法对项目上线速度的影响是无法预计的，同时AI给出的代码不一定是最优方案

程序员很容易陷入反复学习的重复劳动中

#### 今日总结

昨天说到两种ai辅助编程学习的思路，实际上第二种方法也有很大的弊端

ai生成代码的速度和量都是很大的，假如每段代码都要重复审查学习的话

在不了解对应代码的编程语言时，会大幅度拖延项目开发的进度

在今天的实际开发中，我发现了一种折中的较为合适的方法

假如把自己也当成一个智能体，跟ai进行多轮对话，学习ai学习的过程

一开始可以先粗略的描述自己要实现的功能，之后经过不断对话和生成代码，反复学习

之后随着代码量的增加，自己描述逻辑的细节也越来越详细，逐渐做到可以自己编写代码

就这样通过反复学习描述，最终达成掌握对应技能的结果
